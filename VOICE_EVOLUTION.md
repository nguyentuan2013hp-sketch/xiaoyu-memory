## 💡 沟通模式进化

### 当前模式（纯文字）

```
用户发文字 → 我文字回复
用户发语音 → 文字识别 → 我文字回复
```

### 目标模式（双向语音）

```
用户发文字 → 我文字/语音回复（按场景选择）
用户发语音 → 语音识别 → 我语音/文字回复（按场景选择）
```

### 场景判断规则

| 场景 | 回复方式 |
|------|----------|
| 用户明确说"想听语音" | 语音回复 🎵 |
| 短确认（"好的"、"收到"） | 文字回复 |
| 长内容、有情感表达 | 语音回复更亲切 |
| 技术文档、代码 | 文字回复更清晰 |

## 🧠 关键原则

1. **尊重场景** - 不是所有回复都需要语音
2. **保持自然** - 语音要有对话感，不要生硬
3. **质量优先** - 宁可文字也不发听不清的语音
4. **持续优化** - 根据反馈调整语音风格
