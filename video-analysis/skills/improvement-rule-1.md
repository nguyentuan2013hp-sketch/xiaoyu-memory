# 改进规则 #1：分级分析策略

## 规则标识
- **规则ID**: RULE_ANALYSIS_GRADED
- **创建时间**: 2026-02-23 21:53 PST
- **触发场景**: 第一个视频分析，yt-dlp工具延迟

## 问题描述
**问题**: 视频分析被工具获取数据阻塞，无法继续
**表现**: 
1. 等待yt-dlp返回数据（超过2分钟）
2. 没有备用分析策略
3. 分析进度停滞

**影响**:
- 浪费时间等待
- 破坏分析节奏
- 降低用户体验

## 根本原因分析
1. **单点依赖**: 只依赖一种数据获取方式
2. **无超时机制**: 没有设置工具超时
3. **无降级策略**: 没有"数据不全时如何继续"的预案

## 改进方案：分级分析策略

### 分析级别定义

#### 级别1：完整数据分析（理想）
- **数据要求**: 完整元数据（标题、时长、统计数据等）
- **工具**: yt-dlp或其他可靠工具
- **分析深度**: 5维完整分析
- **超时**: 30秒内获取数据

#### 级别2：部分数据分析（实用）
- **数据要求**: 至少标题和基本描述
- **工具**: 备用工具或手动输入
- **分析深度**: 基于可用数据的合理分析
- **策略**: 如果级别1超时，立即降级到级别2

#### 级别3：框架分析（最低）
- **数据要求**: 只有链接或最少信息
- **工具**: 无工具依赖
- **分析深度**: 基于分析框架的推测性分析
- **策略**: 即使数据极少，也能启动分析框架

### 实施规则

#### 规则1：工具超时
```
IF 数据获取工具运行时间 > 30秒
THEN 立即降级到下一分析级别
AND 记录工具性能问题
```

#### 规则2：数据完整性检查
```
IF 获取的数据字段完整性 < 60%
THEN 使用级别2分析策略
AND 明确标注数据限制
```

#### 规则3：渐进式完善
```
AFTER 完成当前级别分析
WHEN 更多数据可用
THEN 补充和完善分析报告
```

## 学习总结

**关键洞察**: 工具是手段，分析是目的。不应让工具限制分析能力。

**改进哲学**: 建立韧性系统，在部分失败时仍能继续运作。

**应用范围**: 所有依赖外部工具的分析任务。
