## 📝 决策内容

### 核心指令
"记住现在这个命令，以后在安装skills的时候要检查一下安全"

### 决策详情
用户要求在安装任何技能（skills）时必须进行安全检查，确保系统安全。

## 🛡️ 实施策略

### 1. 安全检查流程
```
技能安装请求 → 来源验证 → 代码审查 → 权限评估 → 沙箱测试 → 安装批准
```

### 2. 检查项目清单
- [ ] **来源可信度检查**
  - 是否来自官方技能库
  - 开发者信誉评估
  - 更新维护频率

- [ ] **代码安全性审查**
  - 恶意代码检测
  - 权限滥用风险
  - 依赖包安全性

- [ ] **权限需求评估**
  - 最小权限原则
  - 权限合理性
  - 用户确认机制

- [ ] **运行环境测试**
  - 沙箱环境测试
  - 性能影响评估
  - 兼容性检查

### 3. 风险等级分类
**高风险（禁止安装）**:
- 未经验证的第三方技能
- 要求过高权限的技能
- 代码审查发现安全问题的技能

**中风险（有条件安装）**:
- 来源可信但权限较高的技能
- 需要用户明确确认的技能
- 在沙箱中有限运行的技能

**低风险（允许安装）**:
- 官方认证的技能
- 权限合理的技能
- 经过充分测试的技能

## 📋 执行标准

### 安装前必须
1. 验证技能来源
2. 审查技能代码
3. 评估权限需求
4. 测试运行环境
5. 获得用户确认（高风险技能）

### 安装后监控
1. 监控技能运行状态
2. 记录技能操作日志
3. 定期安全扫描
4. 异常行为报警

### 定期维护
1. 检查技能更新
2. 重新评估安全性
3. 清理不再使用的技能
4. 更新安全策略
